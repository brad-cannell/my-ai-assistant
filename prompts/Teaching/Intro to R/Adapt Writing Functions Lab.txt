# Information About Labs

Each course includes at least one lab exercise. Labs allow learners to apply multiple concepts from the course in a more comprehensive and realistic scenario. Labs provide structured guidance while offering more complexity than individual lessons, helping learners bridge the gap between theory and practice. Learners can complete each lab exercise as many times as they like, promoting practice and confidence without penalty.

# Formatting Lab Files

Here are some instructions for formatting the lab files described above:

## Lab Word Document

1. Please name this file Please name this file "Lab - Module Name.docx". For example, "Lab - Writing Functions.docx" for the Writing Functions module.

2. Each task number should be written in bold font.

3. All Questions should be written in a question box (table with one cell). 

4. All R code written in the text should be written in Courier New font with a light gray background.

5. R package names written in the text should be formatted as code (See #4 above).

6. When functions are discussed in the text, they should be formatted as code and empty parentheses added to the end of the function name (See #4 above).

7. Data set and data frame names, variable names, and other key words should be written with **bold** text (e.g., "Please download **chocolate study 1.xls**").

## Lab Quarto File

1. Please name this file "lab_module_name.qmd". For example, "lab_writing_functions.qmd" for the Writing Functions module.

2. The Overview section, each task number, and each question number should be written as a level-one header (e.g., `# Overview` and `# Task 1`).

3. All R code being run, should be written into R code chunks.

4. All R code being discussed in the text (but not run) should be written inline (i.e., placed inside backticks). For example, "Run `install.packages("dplyr")` to install the `dplyr` package.".

5. R package names being discussed in the text should be formatted as code (e.g., "...the `dplyr` package...).

6. When functions are discussed in the text, they should be formatted as code and empty parentheses added to the end of the function name (e.g., "The `tibble()` function can be used to create data frames…").

7. Data set and data frame names, variable names, and other key words should be written with **bold** text (e.g., "Please download **chocolate study 1.xls**").

8. There should be one line break (return) immediately above all headers.

9. There should be one line break (return) immediately above each code chunk. 

10. There should be at least one line break (return) immediately below each code chunk. If the code chunk is immediately followed by a level-one header, then there will be two line breaks.
